# LLM API 聚合平台全面调查报告

**报告编制人**: Yore  
**编制日期**: 2026-04-16  
**调查范围**: 全球LLM API聚合/中转平台的竞争格局、盈利模式、存在价值及未来前景

---

## 一、执行摘要

LLM API聚合平台是连接大模型厂商与终端开发者的中间层服务，通过统一接口聚合多家模型API，降低开发者的接入成本和切换门槛。随着DeepSeek等开源模型引发的价格战，这一赛道正经历剧烈洗牌。

**核心结论**:

| 维度 | 判断 |
|------|------|
| **短期(1-2年)** | 存活但利润承压，价格战压缩中间层利润空间 |
| **中期(3-5年)** | 分化——有自研推理基础设施的平台存活，纯中间商淘汰 |
| **长期(5年+)** | 转型为AI Infra平台或垂直解决方案商，纯聚合模式消亡 |
| **投资价值** | ⭐⭐⭐ 中等，需甄别有技术壁垒的玩家 |

## 三、重点平台详细分析

### 3.1 硅基流动 (SiliconFlow) — 推理云+聚合型

#### 基本信息
- **成立时间**: 2023年
- **创始人**: 袁进辉（OneFlow创始人）
- **融资**: 2024年完成数亿元A轮融资，估值超30亿元
- **投资方**: 华人文化、光源资本等
- **定位**: 国内最大的AI推理云服务商之一

#### 核心竞争力
1. **自研推理引擎**: 基于OneFlow技术积累，推理效率业界领先
2. **自有GPU集群**: 拥有大规模A100/H800集群，不依赖第三方
3. **开源生态布局**: 
   - 开源OneDiff（图像推理加速）
   - 开源SF-VLLM（大模型推理加速）
   - 积极参与开源社区建设
4. **价格优势**: 基于自研推理优化，单位token成本低于官方API
5. **模型覆盖**: 聚合主流开源模型（Qwen、DeepSeek、Llama、GLM等）

#### 定价策略（参考）
| 模型 | 硅基流动定价 | 官方定价 | 折扣 |
|------|------------|---------|------|
| DeepSeek-V3 | ¥1/M tokens | ¥2/M tokens | 50% |
| Qwen-72B | ¥4/M tokens | ¥8/M tokens | 50% |
| Llama-3.1-70B | ¥4/M tokens | — | — |
| GLM-4 | ¥5/M tokens | ¥15/M tokens | 67% |

#### 盈利模式
- **推理服务差价**: 自有GPU集群 + 推理优化 → 单位成本低于市场价 → 赚取差价
- **GPU云租赁**: 提供裸GPU和容器化推理服务
- **企业定制**: 为大客户提供私有化部署和定制推理方案
- **生态溢价**: 开源工具引流 → 付费推理服务转化

#### 竞争优势总结
- ✅ 自研推理引擎，技术壁垒高
- ✅ 自有算力，成本可控
- ✅ 开源生态建设，用户粘性强
- ✅ 国内合规，无需VPN
- ⚠️ 依赖开源模型生态，闭源模型接入受限
- ⚠️ GPU资本开支巨大，盈利周期长

### 3.2 OpenRouter — 模型市场型

#### 基本信息
- **成立时间**: 2023年
- **定位**: 全球最大的LLM API聚合市场
- **模式**: 不自建推理，纯聚合+路由

#### 核心竞争力
1. **模型覆盖最广**: 聚合100+模型，包括GPT-4o、Claude、Gemini等闭源模型
2. **统一接口**: OpenAI兼容格式，一行代码切换模型
3. **智能路由**: 根据成本、延迟、质量自动选择最优模型
4. **透明定价**: 明确标注每个模型的加价比例
5. **社区生态**: 用户评分、模型排行、使用统计

#### 定价机制
- 在官方价格基础上加收 **5-15%** 服务费
- 部分模型提供比官方更低的批量价
- 免费模型（如Llama）不收费

#### 盈利模式
- **API差价**: 在官方价格基础上加收服务费
- **企业计划**: 提供SLA保障、优先路由等增值服务
- **数据价值**: 模型使用数据反馈给模型厂商（匿名化）

#### 竞争优势总结
- ✅ 模型覆盖最全，闭源模型接入
- ✅ 切换成本极低，开发者友好
- ✅ 社区生态活跃
- ❌ 无自研推理，利润空间薄
- ❌ 依赖上游模型厂商，议价能力弱
- ❌ 国内访问受限

### 3.3 Fireworks AI — 推理云+聚合型

#### 基本信息
- **成立时间**: 2022年
- **创始人**: 前Meta工程师
- **融资**: 2024年完成B轮融资，估值约5.52亿美元
- **定位**: 高性能开源模型推理云

#### 核心竞争力
1. **极致推理速度**: 基于自研推理引擎，首token延迟<100ms
2. **LoRA微调服务**: 支持在线微调开源模型，无需自建GPU
3. **函数调用优化**: 专门优化了function calling性能
4. **企业级SLA**: 提供99.9%可用性保障

#### 盈利模式
- **推理服务**: 按token计费，价格低于官方
- **LoRA微调**: 按训练时长计费
- **企业方案**: 私有化部署、定制推理

#### 竞争优势总结
- ✅ 推理速度业界领先
- ✅ 微调服务差异化
- ✅ 企业客户粘性强
- ⚠️ 仅支持开源模型
- ⚠️ 面临硅基流动等竞品压力

### 3.4 Together AI — 推理云+开发者平台型

#### 基本信息
- **成立时间**: 2022年
- **融资**: 2024年完成1.06亿美元A轮融资，估值12.5亿美元
- **投资方**: Salesforce Ventures、NVIDIA等
- **定位**: AI开发者全栈平台

#### 核心竞争力
1. **全栈工具链**: 推理→微调→训练→部署一站式服务
2. **自研推理引擎**: 基于FlashAttention等优化技术
3. **模型训练服务**: 提供从LoRA到全参数微调的能力
4. **Embeddings服务**: 提供多种嵌入模型
5. **开源贡献**: 积极贡献FlashAttention、RedPajama等开源项目

#### 盈利模式
- **推理服务**: 按token计费
- **训练/微调**: 按GPU时长计费
- **企业方案**: 私有化部署
- **开源商业化**: 开源工具引流→付费服务转化

#### 竞争优势总结
- ✅ 全栈能力，用户生命周期价值高
- ✅ NVIDIA投资，算力获取有优势
- ✅ 开源生态建设扎实

### 3.5 Groq — 推理硬件+云服务型

#### 基本信息
- **成立时间**: 2016年（原做TPU，后转型LPU）
- **创始人**: Jonathan Ross（前Google TPU核心成员）
- **融资**: 2024年完成6.4亿美元D轮融资，估值28亿美元
- **定位**: LPU（语言处理单元）芯片+推理云

#### 核心竞争力
1. **自研LPU芯片**: 专为LLM推理设计，速度远超GPU
2. **极致推理速度**: Llama-70B可达300+ tokens/s（GPU约50-80 tokens/s）
3. **确定性延迟**: LPU架构保证推理延迟的确定性
4. **开源生态**: 提供免费API额度吸引开发者

#### 盈利模式
- **推理云服务**: 按token计费（目前以免费/低价获客为主）
- **硬件销售**: LPU芯片/服务器销售
- **企业部署**: 私有化LPU集群方案

#### 竞争优势总结
- ✅ 自研芯片，技术壁垒最高
- ✅ 推理速度碾压GPU方案
- ✅ 确定性延迟，适合实时应用
- ❌ 芯片产能有限，规模化困难
- ❌ 仅支持推理，不支持训练
- ❌ 内存容量受限，大模型支持有限

### 3.6 国内API中转站（API2D、OhMyGPT等）— 纯API中转型

#### 基本信息
- **形态**: 个人或小团队运营的API代理服务
- **核心功能**: 帮国内用户绕过网络限制访问GPT/Claude等API
- **代表**: API2D、OhMyGPT、AI.LS、API2GPT等

#### 盈利模式
- **汇率差+加价**: 在官方美元价格基础上加价30-100%
- **充值优惠**: 利用批量购买和企业折扣获取低价
- **多级代理**: 发展下级代理赚取差价

#### 风险分析
| 风险类型 | 严重程度 | 说明 |
|---------|---------|------|
| 合规风险 | 🔴 极高 | 未经授权代理境外API，可能违反数据出境法规 |
| 账号封禁 | 🔴 极高 | OpenAI等厂商频繁封禁代理账号 |
| 价格战 | 🟡 高 | 进入门槛低，利润持续被压缩 |
| 替代风险 | 🔴 极高 | 国产模型（DeepSeek、Qwen）性能逼近，需求萎缩 |

#### 竞争优势总结
- ✅ 解决国内用户访问痛点
- ✅ 门槛低，启动快
- ❌ 无技术壁垒，随时可被替代
- ❌ 合规风险极高
- ❌ 随国产模型崛起，存在价值快速下降

## 五、盈利模式深度分析

### 5.1 四种盈利模式详解

#### 模式一：推理差价（Infra Play）

**代表**: 硅基流动、Fireworks AI、Together AI、Groq

```
收入 = Σ(用户调用量 × 单价)
成本 = GPU折旧 + 电力 + 带宽 + 运维
利润 = 收入 - 成本

关键指标: GPU利用率 × 推理效率
```

**利润拆解**:
| 成本项 | 占比 | 说明 |
|--------|------|------|
| GPU折旧 | 40-50% | A100约$15K/H100约$30K，3年折旧 |
| 电力 | 15-20% | 单卡300-700W |
| 带宽 | 5-10% | 推理响应的带宽成本 |
| 运维人力 | 5-10% | 基础设施运维团队 |
| **毛利率** | **25-40%** | 依赖GPU利用率和推理优化 |

**核心逻辑**: 自研推理引擎 → 单卡吞吐量提升2-5倍 → 单位成本降低 → 低于官方定价 → 赚取差价

**关键风险**: GPU资本开支巨大，需要持续融资支撑扩张

#### 模式二：API加价（Reseller Play）

**代表**: OpenRouter、API2D、OhMyGPT

```
收入 = Σ(用户调用量 × 加价后单价)
成本 = Σ(用户调用量 × 官方单价) + 运营成本
利润 = 收入 - 成本

关键指标: 用户量 × 加价率
```

**利润拆解**:
| 成本项 | 占比 | 说明 |
|--------|------|------|
| 上游API成本 | 85-95% | 按官方价格采购 |
| 运营成本 | 3-5% | 服务器、带宽 |
| 人力 | 2-3% | 小团队运营 |
| **毛利率** | **5-15%** | 加价空间有限 |

**核心逻辑**: 聚合流量 → 批量采购折扣 → 加价出售 → 赚取差价

**关键风险**: 上游厂商降价直接压缩利润，无议价能力

#### 模式三：平台抽成（Marketplace Play）

**代表**: Replicate、Hugging Face

```
收入 = Σ(模型调用量 × 抽成比例)
成本 = GPU基础设施 + 平台运营
利润 = 收入 - 成本

关键指标: 模型数量 × 开发者活跃度 × 抽成率
```

**利润拆解**:
- 抽成比例通常为 **15-30%**
- 毛利率约 **20-35%**

**核心逻辑**: 构建模型市场 → 吸引模型创作者和用户 → 双边网络效应 → 抽成

**关键风险**: 冷启动困难，需要同时吸引供需双方

#### 模式四：全栈服务（Platform Play）

**代表**: Together AI、硅基流动（企业版）

```
收入 = 推理服务 + 训练/微调服务 + 企业方案 + 咨询
成本 = GPU基础设施 + 人力 + 运营
利润 = 收入 - 成本

关键指标: 客户生命周期价值(LTV)
```

**利润拆解**:
| 服务 | 毛利率 | 说明 |
|------|--------|------|
| 推理API | 25-40% | 标准化服务 |
| 微调服务 | 40-60% | 技术含量高 |
| 企业方案 | 50-70% | 高度定制 |
| 咨询服务 | 60-80% | 知识密集型 |

**核心逻辑**: 低毛利推理引流 → 高毛利服务变现 → 企业客户锁定

### 5.2 DeepSeek价格冲击的影响

2025年1月，DeepSeek-V3以极低价格发布API（¥1/M input tokens），对整个聚合平台生态产生深远影响：

| 影响维度 | 具体表现 |
|---------|---------|
| **价格锚点下移** | 所有模型的定价锚点被拉低，聚合平台加价空间被压缩 |
| **推理效率要求提升** | 只有推理效率足够高的平台才能以低于DeepSeek的价格提供服务 |
| **纯中转站加速消亡** | 当官方API已经足够便宜，中转站的加价变得不可接受 |
| **差异化竞争加剧** | 平台必须提供除价格外的其他价值（速度、微调、生态） |

**关键数据**: DeepSeek-V3发布后，OpenRouter上DeepSeek模型的使用量在两周内跃升至第一，占比超过30%。

## 六、存在价值分析

### 6.1 聚合平台解决了什么问题？

#### 问题一：模型碎片化
```
开发者痛点:
├── GPT-4o → OpenAI API格式
├── Claude → Anthropic API格式  
├── Gemini → Google API格式
├── Qwen → 阿里云API格式
└── DeepSeek → 独立API格式

聚合平台价值: 一个接口，统一格式，一行代码切换
```

#### 问题二：推理成本高
```
官方API定价 vs 聚合平台定价:
├── DeepSeek-V3: 官方¥2/M → 硅基流动¥1/M (省50%)
├── Qwen-72B: 官方¥8/M → 硅基流动¥4/M (省50%)
└── Llama-70B: 无官方API → 聚合平台¥4/M (提供可用性)
```

#### 问题三：推理部署门槛
```
自建推理的门槛:
├── GPU采购: A100×8 = $120K+
├── 推理引擎: vLLM/TensorRT-LLM部署复杂
├── 运维: 7×24监控、扩缩容
└── 成本: 人力+硬件+电力

聚合平台价值: API调用即可，零运维
```

#### 问题四：模型选择困难
```
开发者困惑:
├── 100+模型如何选择？
├── 哪个模型性价比最高？
├── 如何A/B测试不同模型？
└── 如何根据任务自动路由？

聚合平台价值: 模型排行、智能路由、A/B测试
```

## 七、未来前景判断

### 7.1 行业趋势

#### 趋势一：API价格持续下降
- DeepSeek已将价格锚点拉至极低水平
- 开源模型性能逼近闭源，价格优势明显
- 推理成本每年下降约50-70%（硬件+算法双重优化）
- **影响**: 纯靠API差价的利润空间持续收窄

#### 趋势二：推理基础设施化
- LLM推理正在从"服务"变为"基础设施"
- 类比云计算发展史：IaaS → PaaS → SaaS
- 推理服务将走向白热化竞争，最终只剩2-3家规模玩家
- **影响**: 小规模推理云将被兼并或淘汰

#### 趋势三：模型厂商直销化
- OpenAI、Anthropic等厂商正在加强直销能力
- 企业客户更倾向直接与模型厂商签约
- 模型厂商提供越来越完善的API管理工具
- **影响**: 聚合平台的中间层价值被削弱

#### 趋势四：端侧推理崛起
- Apple Intelligence、高通AI引擎推动端侧推理
- 小模型（7B以下）可在手机/PC本地运行
- 隐私敏感场景倾向本地推理
- **影响**: 部分推理需求从云端迁移到端侧

#### 趋势五：垂直化与专业化
- 通用聚合平台的差异化越来越难
- 垂直领域（医疗、法律、金融）的专用推理服务兴起
- **影响**: 通用平台需向垂直化转型或被垂直方案替代

### 7.2 各类平台未来预测

| 平台类型 | 1-2年 | 3-5年 | 5年+ |
|---------|-------|-------|------|
| **纯API中转站** | 大量倒闭 | 基本消亡 | 不复存在 |
| **推理云+聚合** | 价格战激烈，头部集中 | 2-3家规模玩家胜出 | 转型为AI Infra平台 |
| **模型市场** | 用户增长，但变现困难 | 被推理云整合或独立存活 | 成为AI领域的"App Store" |
| **开发者平台** | 差异化竞争，稳步增长 | 形成稳定生态 | 成为AI开发标准工具 |

### 7.3 最终判断：聚合平台是否会继续存在？

**结论：会存在，但形态将发生根本性变化。**

#### 会消亡的：
- ❌ 纯API中转站（无技术壁垒，合规风险高）
- ❌ 仅靠API差价的简单聚合（利润空间归零）
- ❌ 无差异化优势的跟风平台

#### 会存活的：
- ✅ 有自研推理基础设施的平台（硅基流动、Fireworks）
- ✅ 有硬件壁垒的平台（Groq）
- ✅ 有开发者生态的平台（Together AI、Replicate）
- ✅ 有独特定位的模型市场（OpenRouter）

#### 会新生的：
- 🆕 AI Agent编排平台（不只是模型调用，而是Agent编排）
- 🆕 垂直行业推理服务（医疗影像AI推理、法律AI推理等）
- 🆕 边缘推理聚合平台（协调端侧+云端推理）

### 7.4 类比分析：聚合平台的历史参照

| 行业 | 中间层平台 | 最终命运 | 对LLM聚合的启示 |
|------|-----------|---------|----------------|
| 云计算 | Cloud Foundry/Heroku | 被IaaS厂商整合 | 推理云可能被大厂整合 |
| 支付 | Stripe/支付宝 | 独立存活，构建生态 | 聚合平台需构建生态壁垒 |
| 旅行 | 携程/Booking | 独立存活，但利润承压 | 统一接口的价值持续存在 |
| 通信 | Twilio | 独立存活，转向CPaaS | 从通道商转向平台商 |

**核心启示**: 中间层平台能否存活，取决于能否从"通道"进化为"生态"。

## 九、总结

### 9.1 一句话结论

> **LLM API聚合平台不会消亡，但会从"API中转站"进化为"AI基础设施平台"——只有拥有技术壁垒和生态护城河的玩家才能存活。**

### 9.2 核心洞察

1. **价格战是催化剂，不是终结者**: DeepSeek的价格冲击加速了行业洗牌，但也验证了推理效率优化的价值
2. **推理基础设施化是大趋势**: LLM推理正在成为像CDN一样的基础设施，最终只有规模玩家能胜出
3. **中间层的价值在于"减法"而非"加法"**: 聚合平台的价值不是加价，而是帮开发者减少接入成本、运维成本、选择成本
4. **合规是底线**: 在中国市场，合规性是生存的前提
5. **从聚合到编排**: 未来的平台不只是"模型路由器"，而是"AI工作流编排器"
